## 2. 4개 레이어 구조

### 레이어 1 — 입력 (사용자가 정의)
| 입력 | 방식 | 저장 |
|------|------|------|
| 섹터 선택 | 드롭다운 (반도체/로봇/2차전지…) | sectors/{name}.json |
| 섹터별 종목 목록 | 사용자가 종목코드 추가 | 위 json 안에 |
| 기업→기능블록 매핑 | 사용자가 분류 | 위 json 안에 |
| 공급관계 (A→B, 의존도) | 사용자 입력 UI | 위 json 안에 |
| 시장점유율 | 사용자 입력/추정 | 위 json 안에 |

→ 섹터 하나 = json 파일 하나. 사용자가 만들고 키워가는 자산.

### 레이어 2 — 데이터 (자동/반자동 혼합)
- 자동: 시세·시가총액·매출·영업이익 (yfinance)
- 반자동: 공급관계·점유율·기능블록 (사용자 입력, 위에서)
- 나중: AI 독해가 공급관계·기능블록 "초안" 생성 → 사용자 검수

### 레이어 3 — 엔진 (이미 완성, 손 안 댐)
- 그래프 구성 → 곡괭이 점수 → 4분면 분류
- **유일한 수정**: core/graph.py의 Company에 functional_block 필드 추가

### 레이어 4 — 출력 (새 화면 3개)
1. **기능블록 지도**: 섹터를 블록으로 쪼개고 블록별 지배 기업 표시
2. **대장주 랭킹**: 매출+영익+점유율 종합 점수 순
3. **곡괭이·연결도**: 숨은 곡괭이 4분면 + 공급관계 그래프 시각화

## 3. 데이터 구조 (섹터 json 스키마)

```json
{
  "sector": "로봇",
  "companies": [
    {
      "code": "277810",
      "name": "레인보우로보틱스",
      "functional_blocks": ["판단", "골격"],
      "market_share": 0.15,
      "is_final_product": true
    }
  ],
  "functional_blocks": [
    "두뇌", "시야", "관절", "전력", "음성", "연결", "판단", "골격"
  ],
  "supply_relations": [
    {"from": "부품사코드", "to": "완성품코드",
     "block": "관절", "part": "감속기", "dependency": 0.6}
  ]
}
```

→ 시세·재무는 여기 저장 안 함 (yfinance로 실시간 조회). 구조 정보만 저장.

## 4. 대장주 점수 (확정된 공식)

```
대장주 점수 = 0.35×매출순위 + 0.35×영업이익순위 + 0.30×점유율
(각 지표는 섹터 내에서 0~1로 정규화)
```
- 매출·영업이익: yfinance 자동
- 점유율: 사용자 입력 (없으면 0.30 가중치 빼고 재정규화)

곡괭이 점수와는 별개 축. 대장주 = "지금 큰 기업", 곡괭이 = "구조적으로 중요한 기업".
둘을 같이 보여줘서 "대장주지만 곡괭이는 아님" vs "작지만 핵심 곡괭이" 구분.

## 6. AI 독해 엔진 설계 (STEP 6 상세)

목적: 비정형 텍스트(뉴스·공시)를 읽어 밸류체인 구조를 "정확하게" 추출.
**직접 딥러닝 학습 아님.** 이미 학습된 LLM의 독해력을 빌림.

작동:
1. 도구가 특정 섹터 관련 텍스트(공시·뉴스) 수집
2. LLM에게 전달: "아래 텍스트에 명시된 공급관계만 추출하라.
   텍스트에 없는 내용은 지어내지 마라. 추출형식: from/to/part/근거문장"
3. LLM이 구조화 결과 반환 (근거 문장 포함)
4. 사용자가 각 추출 항목을 검수/승인 → 그래프에 반영

정확도 보장 2원칙 (절대 준수):
- **근거 강제**: LLM 지식이 아니라 제공된 텍스트 안에서만 추출. 근거문장 필수.
- **사람 검수**: LLM 추출은 "초안". 확정은 사용자.

비용: 섹터당 1회성 분석 (구조는 자주 안 바뀜). 실시간 호출 아님.
사용량 상한 설정으로 통제.

## 7. 관통 원칙 (모든 단계에서 준수)

- 객관적 숫자(시세·재무)는 자동. 해석·분류(블록·공급관계·점유율)는 사용자 확정.
- AI는 "초안 생성 노동"만. 판단·확정은 사람. 근거 없는 출력 금지.
- 예측하지 않음. 현재 구조만 이해.
- 기존 엔진 3개 재사용. 바퀴 재발명 금지.
- 사용자는 ML·자료구조 학생. 설명하며 진행. 간결·직접적으로.
- 투자 자문 아님. 구조 분석 도구.
